  PART A — AI COACH  

1. Feature Overview
GOAL
After every completed workout, FitTribe shows a personalised AI insight on the Summary screen. The insight uses the user's actual profile data and session history — making it feel like advice from a real coach who knows them.

This is Feature 1 of the AI roadmap — the MVP AI feature. It works from day 1 of a user's account and gets smarter with every session they log.

1.1  Why This Feature
	•	No Indian gym app has personalised AI tied to real session data
	•	Hevy and Strong have no AI — only generic tips
	•	Works from day 1, gets smarter with every session
	•	Cost: ₹0.01 per session — negligible at any scale
	•	Fails silently — core app never breaks if OpenAI is down

1.2  What the User Sees
On the Summary screen, after finishing a workout, a card appears below the stats grid. Two versions — Day 1 vs Session 5+:

🤖  AI Coach  ·  Session 1

"At 55 kg body weight, 10 kg shoulder press is strong for a beginner
woman. Focus on form over weight for the first 4 weeks — injuries
at this stage set beginners back by months."

🤖  AI Coach  ·  Session 5+

"Your shoulder press has gone from 8 kg to 12 kg in 4 sessions —
excellent progression. Weekly volume is up 38%. Make sure you're
eating enough protein to support this growth rate."

4. The Prompt — Most Important Section
KEY INSIGHT
The API call is 20 lines. The prompt is the product. A bad prompt gives generic tips. A good prompt gives advice that feels like a personal trainer who has been watching for weeks.

4.1  Full Prompt Template
SYSTEM PROMPT

You are an expert fitness coach inside FitTribe, an Indian gym app.
Be specific — never generic. Always reference their actual numbers.
Max 2-3 sentences. Tone: knowledgeable friend. No emojis.

USER PROMPT (built per session)

USER PROFILE:
Name: {display_name} | Age: {age} | Gender: {gender}
Height: {height_cm}cm | Weight: {weight_kg}kg

TODAY'S SESSION ({date}):
Duration: {duration} min | Total volume: {total_volume_kg} kg
- {exercise_name}: {weight}kg x {reps} reps x {sets} sets
- {exercise_name}: {weight}kg x {reps} reps x {sets} sets
PRs today: {pr_list or "none"}

[INCLUDE ONLY IF session_count >= 3]
LAST 3 SESSIONS:
  {date}: volume {X}kg | {exercise} {kg}kg ...
  {date}: volume {X}kg | {exercise} {kg}kg ...

[END BLOCK]

[IF session_count < 3]

5.3  AiService.java — Core Logic
@Service @Slf4j @RequiredArgsConstructor
public class AiService {
  @Value("${openai.api-key}") private String apiKey;
  @Value("${openai.model}")   private String model;
  @Value("${openai.max-tokens}") private int maxTokens;
  @Value("${openai.insight-min-sets}") private int minSets;
  private final WorkoutSessionRepository sessionRepo;
  private final UserRepository userRepo;
  private final RestTemplate restTemplate;
  private static final String URL = "https://api.openai.com/v1/chat/completions";

  @Async
  public void generateAndSaveInsight(UUID userId, UUID sessionId) {
    try {
      var session = sessionRepo.findById(sessionId).orElseThrow();
      if (session.getSetLogs().size() < minSets) return;
      String prompt  = buildPrompt(userId, session);
      String insight = callOpenAI(prompt);
      if (insight != null) {
        session.setAiInsight(insight);
        session.setAiInsightGeneratedAt(Instant.now());
        sessionRepo.save(session);
      }
    } catch (Exception e) {
      log.warn("AI insight failed for {}: {}", sessionId, e.getMessage());
    }
  }
  // buildPrompt() and callOpenAI() — see Section 4.1 for prompt logic
}

5.4  Wire Into Session Finish
// In SessionController.java — add ONE line after session is marked COMPLETED:
aiService.generateAndSaveInsight(userId, id);  // async, non-blocking

5.5  API Endpoint
GET  /api/v1/ai/insight/{sessionId}   — returns 200 with insight or 204 if not ready

  PART B — EMOTIONAL HOOKS  

8. What is an Emotional Hook
An emotional hook is a moment in an app that makes you feel something — pride, fear, urgency, or belonging — strongly enough that you take action or come back tomorrow.

It is not a feature. It is a feeling triggered by a specific design decision.

THE TEST
Does the user feel something when they see this? If the answer is no, it is information. Not a hook.

8.1  Why Your Current Feed is Weak
Right now your feed shows:

Group Feed

Rahul crushed Leg Day — 6,200 kg volume, new squat PR
🦵 Leg Day   🏆 95 kg squat   🔥 22-streak
🔥 8    💪 3

This is information. It tells you what Rahul did. It does not make you feel anything about it. You read it and scroll past.

The difference is not the data. It is personal relevance. The feed needs to show what things mean FOR THE PERSON READING IT — not just what happened.

10. The 5 Emotional Hooks for FitTribe
Hook 1 — Streak Anxiety
FEELING
Mild dread. The fear of losing something you worked hard for.

Notification — fires at 8pm if user has not logged
Push Notification

🔥 Your 17-day streak ends in 4 hours.
Mumbai crew needs 1 more person today.
That person could be you.

Home screen card — changes at 8pm for users who have not logged
Home Screen — Streak at Risk State

⚠️  Streak at risk
17 days. Tonight by midnight.

[Log workout →]

The card border pulses. The card animates gently when the app opens if less than 2 hours remain.

What to build
	•	New home state: streakAtRisk — shows when isActive=false, hasStreak=true, time > 8pm
	•	Push notification scheduled daily at 8pm via Spring @Scheduled
	•	Pulsing CSS animation on the at-risk card border
	•	New AppContext state: streakAtRisk boolean

Hook 2 — The Poke That Hurts a Little
FEELING
Playful shame. The social awkwardness of being called out by friends.

Notification received by the person who has not logged
Push Notification

Priya poked you 👋
She has already logged today.
You haven't. Mumbai crew is watching.

Feed card for a member who has not logged
Feed Card — Not Logged State

Vikas — No activity today
🧊 Last logged 2 days ago

[Poke 👋]   Already poked by 2 people

"Already poked by 2 people" — this is the hook. Nobody wants to be poked by 3 friends and still not go to the gym. Social visibility creates action.

What to build
	•	Track poke count per member per day in a new DB column: group_members.poke_count_today
	•	Show poke count on feed card if > 0
	•	Push notification copy change — personalise with poker name and their logged status
	•	Reset poke count at midnight daily

Hook 3 — The Near Miss
FEELING
Competitive urgency. The specific pain of almost winning.

Borrowed from gaming — finishing #2 hurts more than #10. The near miss is more motivating than a comfortable lead or a comfortable loss.

Leaderboard — Near Miss View

🏋️  This week's effort leaderboard

1.  Rahul     94 pts  ████████████
2.  You       88 pts  ██████████░░  ← only 6 pts behind
3.  Priya     71 pts

You are 6 points behind the top spot.
One solid session could flip this.

Show the gap in small, specific units. Not "you are losing" — "you are THIS CLOSE to winning." The specificity of the gap is the hook.

What to build
	•	Calculate gap to #1 in real time from effort scores
	•	Show gap prominently below user's row: "X points behind the top spot"
	•	Only show if gap is <= 15 points — larger gaps feel hopeless, not motivating
	•	Add message: "One solid session could flip this" when gap <= 10

Hook 4 — The PR Moment
FEELING
Pure pride. The specific joy of hitting a personal best.

Right now PRs are a small chip in the feed. That is not enough. When a user hits a PR, interrupt the logging screen — stop everything.

PR Celebration — Full Screen Interrupt

                    🏆

         NEW PERSONAL RECORD

           Bench Press
         85 kg × 5 reps

       Up from 82.5 kg last week

  [Share to Mumbai crew →]   [Continue workout]

Group feed PR card — looks visually different from regular workout cards
🏆  PR ALERT

Arjun just hit a new bench press record
85 kg — up 3% from last week

[🔥 React]   [💪 React]   [Send a message]

The gold border and PR ALERT label make group members feel like they witnessed something. They want to react.

What to build
	•	PR detection already exists in set_logs.is_pr — surface it dramatically
	•	Full-screen PR modal in Logging.jsx — fires when is_pr is true on any set tick
	•	Confetti animation using a lightweight library (canvas-confetti — 3kb)
	•	Auto-post to group feed with gold border card variant
	•	Different feed card component: PRFeedCard — gold border, 🏆 header, larger text

Hook 5 — Group Milestone Drama
FEELING
Collective pride. The specific feeling of being part of something that succeeded together.

When the group hits a streak milestone — 7, 14, 30 days — do not just update a number. Make it a moment.

Notification — every group member at the same time
Push Notification — Milestone

Mumbai crew just hit 14 days 🔥
All 6 members contributed this week.
You are the longest-running group on FitTribe right now.

"The longest-running group on FitTribe right now" — this line is the hook. Competitive, specific, exclusive. People screenshot this.

Group screen banner — temporary, shown for 24 hours after milestone
Group Screen — Milestone Banner

🔥  14-day group streak!
Mumbai crew · All 6 logged ✓
Longest active streak on FitTribe

What to build
	•	Milestone check in GroupStreakService after every session logged
	•	Milestones: 7, 14, 21, 30, 60, 100 days
	•	Push notification to all group members simultaneously via FCM
	•	Group rank query: SELECT COUNT(*) FROM groups WHERE streak > this_group_streak
	•	Banner component in Group.jsx — visible for 24 hours after milestone hit

12. The Core Principle
RULE
Make it about the user, not about what someone else did. Same data. Different framing. That is what an emotional hook is.
